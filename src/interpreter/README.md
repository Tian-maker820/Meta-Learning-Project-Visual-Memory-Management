# 解释器模块

本目录包含时间旅行调试器的核心解释器实现。

## 模块概述

解释器模块实现了从源代码解析到执行的全过程，包括词法分析、语法分析、抽象语法树构建、解释执行和状态管理。

## 核心类

### 1. Value (值类型系统)
- **文件**: `value.h`, `value.cpp`
- **功能**: 使用`std::variant`实现多类型值系统
- **支持类型**: `int`, `float`, `bool`, `Array`（数组）
- **操作**: 算术运算、比较运算、逻辑运算、数组操作
- **设计模式**: 值对象模式

### 2. Tokenizer (词法分析器)
- **文件**: `tokenizer.h`, `tokenizer.cpp`
- **功能**: 将源代码字符串转换为Token序列
- **支持的Token**: 关键字、标识符、字面量、运算符、分隔符
- **算法**: 基于有限状态机的扫描器

### 3. Parser (语法分析器)
- **文件**: `parser.h`, `parser.cpp`
- **功能**: 使用递归下降法解析Token序列，构建抽象语法树
- **语法规则**: 实现简单C语言子集的文法
- **错误处理**: 语法错误报告和错误恢复

### 4. AST (抽象语法树)
- **文件**: `ast.h`, `ast.cpp`
- **功能**: 定义AST节点类型和执行语义
- **节点类型**:
  - 表达式节点: `LiteralNode`, `VariableNode`, `BinaryOpNode`, `UnaryOpNode`, `ArrayAccessNode`
  - 语句节点: `VariableDeclarationNode`, `AssignmentNode`, `PrintNode`, `IfNode`, `WhileNode`, `BlockNode`
- **设计模式**: 访问者模式（通过虚函数实现）

### 5. ExecutionContext (执行上下文)
- **文件**: `executioncontext.h`, `executioncontext.cpp`
- **功能**: 管理变量作用域、控制台输出和当前执行状态
- **特性**: 支持嵌套作用域、变量查找、数组元素访问

### 6. ProgramState (程序状态快照)
- **文件**: `programstate.h`, `programstate.cpp`
- **功能**: 保存程序在某个时刻的完整状态
- **包含信息**: 所有变量值、当前行号、控制台输出
- **设计考虑**: 深拷贝实现，确保状态独立性

### 7. Interpreter (解释器主类)
- **文件**: `interpreter.h`, `interpreter.cpp`
- **功能**: 协调各个模块，实现程序执行和时间旅行
- **状态管理**: `IDLE`, `RUNNING`, `PAUSED`, `FINISHED`
- **核心功能**: 执行控制、快照管理、时间旅行跳转

## 执行流程

1. **词法分析**: `源代码` → `Tokenizer` → `Token序列`
2. **语法分析**: `Token序列` → `Parser` → `AST`
3. **解释执行**: `AST` → `Interpreter` → `执行结果`
4. **状态保存**: 每执行一步 → `ProgramState` → `快照列表`
5. **时间旅行**: 用户交互 → `快照索引` → `状态恢复`

## 设计模式应用

### 1. 解释器模式 (Interpreter Pattern)
- **应用**: AST节点执行
- **实现**: 每个AST节点实现`execute()`方法
- **优点**: 易于扩展新的语法结构

### 2. 访问者模式 (Visitor Pattern)
- **应用**: AST遍历和执行
- **实现**: 通过虚函数多态实现
- **优点**: 分离语法结构和执行逻辑

### 3. 备忘录模式 (Memento Pattern)
- **应用**: 程序状态快照
- **实现**: `ProgramState`作为备忘录，`Interpreter`作为原发器
- **优点**: 实现状态保存和恢复，支持时间旅行

### 4. 状态模式 (State Pattern)
- **应用**: 解释器执行状态
- **实现**: `InterpreterState`枚举和状态转换
- **优点**: 简化状态管理逻辑

## 复杂度分析

### 时间复杂度
- **词法分析**: O(n)，n为源代码长度
- **语法分析**: O(n)，递归下降解析
- **执行**: O(m)，m为AST节点数
- **快照保存**: O(k)，k为变量数量

### 空间复杂度
- **Token存储**: O(n)
- **AST存储**: O(m)
- **快照存储**: O(s*k)，s为快照数量，k为变量数量

## 优化建议

### 1. 写时复制 (Copy-on-Write)
- **问题**: 每次快照都深拷贝所有变量，内存占用大
- **解决方案**: 实现COW的`Value`和`ProgramState`
- **预期效果**: 减少内存占用50%以上

### 2. 增量快照
- **问题**: 保存完整状态，冗余数据多
- **解决方案**: 只保存相对于前一个快照的变化
- **预期效果**: 进一步减少内存占用

### 3. 懒求值 (Lazy Evaluation)
- **问题**: 表达式求值可能重复计算
- **解决方案**: 缓存表达式求值结果
- **预期效果**: 提高执行速度

### 4. JIT编译
- **问题**: 解释执行速度慢
- **解决方案**: 将热点代码编译为本地代码
- **预期效果**: 大幅提高执行速度
