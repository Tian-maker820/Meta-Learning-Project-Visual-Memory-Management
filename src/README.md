# GUI界面模块

本目录包含时间旅行调试器的Qt GUI界面实现。

## 模块概述

GUI模块提供了用户界面，包括代码编辑器、变量监控、控制台输出和控制面板，实现了与解释器核心的交互。

## 核心部件

### 1. MainWindow (主窗口)
- **文件**: `mainwindow.h`, `mainwindow.cpp`
- **功能**: 应用程序主窗口，协调各个部件
- **布局管理**: 使用QSplitter实现可调整的布局
- **交互逻辑**: 连接解释器和UI部件

### 2. EditorWidget (代码编辑器)
- **文件**: `editorwidget.h`, `editorwidget.cpp`
- **功能**: 代码编辑和显示
- **特性**:
  - 行号显示
  - 语法高亮（支持C语言子集）
  - 当前行高亮
  - 调试行高亮
- **设计**: 自定义行号区域，简单语法高亮器

### 3. VariableWidget (变量监控)
- **文件**: `variablewidget.h`, `variablewidget.cpp`
- **功能**: 显示程序执行过程中的变量状态
- **界面**: QTableWidget表格显示
- **列**: 变量名、值、类型
- **更新**: 根据程序状态实时更新

### 4. ConsoleWidget (控制台输出)
- **文件**: `consolewidget.h`, `consolewidget.cpp`
- **功能**: 显示程序执行输出
- **特性**:
  - 支持普通输出和错误输出（不同颜色）
  - 自动滚动到底部
  - 清空功能
- **设计**: QPlainTextEdit + 控制按钮

### 5. ControlWidget (控制面板)
- **文件**: `controlwidget.h`, `controlwidget.cpp`
- **功能**: 程序执行控制和状态显示
- **组件**:
  - 控制按钮: 运行、单步、重置、暂停
  - 时间轴滑块: 时间旅行控制
  - 状态标签: 显示执行状态和步骤信息
- **状态管理**: 根据解释器状态更新按钮状态

## 界面布局

```
+---------------------------------------------------+
| 菜单栏                                            |
+-------------------+-------------------------------+
|                   |                               |
|                   |        变量监控               |
|    代码编辑器     +-------------------------------+
|                   |                               |
|                   |        控制台输出             |
|                   |                               |
+-------------------+-------------------------------+
| 运行 单步 重置 暂停 | [=======时间轴滑块=======]   |
| 状态: 就绪  步骤: 0/0                            |
+---------------------------------------------------+
```

### 布局特点
1. **可调整分割**: 使用QSplitter，用户可以调整各区域大小
2. **响应式设计**: 窗口大小变化时自动调整布局
3. **直观操作**: 控制按钮和时间轴滑块位置明显

## 交互设计

### 1. 代码编辑
- **实时编辑**: 代码修改后自动重置执行状态
- **语法高亮**: 关键字、类型、数字、字符串、注释不同颜色
- **行号显示**: 方便调试和定位

### 2. 执行控制
- **运行**: 连续执行整个程序
- **单步**: 逐条语句执行
- **暂停**: 暂停当前执行
- **重置**: 清除所有状态，重新开始

### 3. 时间旅行
- **滑块控制**: 拖动滑块选择历史步骤
- **实时更新**: 滑块移动时实时更新变量和代码高亮
- **状态恢复**: 跳转到指定步骤时恢复完整程序状态

### 4. 状态反馈
- **变量监控**: 实时显示当前步骤的变量值
- **控制台输出**: 显示程序执行输出
- **状态提示**: 显示当前执行状态（就绪、运行中、已暂停、已完成）
- **步骤信息**: 显示当前步骤/总步骤数

## 设计模式应用

### 1. MVC模式 (Model-View-Controller)
- **Model**: `Interpreter`（解释器核心）
- **View**: 各个UI部件（EditorWidget、VariableWidget等）
- **Controller**: `MainWindow`（主窗口协调）

### 2. 观察者模式 (Observer Pattern)
- **应用**: 解释器状态变化通知UI更新
- **实现**: 回调函数机制
- **优点**: 解耦解释器和UI，支持异步更新

### 3. 组合模式 (Composite Pattern)
- **应用**: UI部件层次结构
- **实现**: QWidget派生类组合
- **优点**: 统一的部件接口，易于扩展

### 4. 策略模式 (Strategy Pattern)
- **应用**: 语法高亮算法
- **实现**: `SimpleHighlighter`类
- **优点**: 易于替换高亮算法

## 性能考虑

### 1. UI响应性
- **异步更新**: 解释器执行在后台线程，UI保持响应
- **增量更新**: 只更新变化的部分，减少UI重绘
- **防抖动**: 滑块拖动时适当延迟更新，避免频繁刷新

### 2. 内存管理
- **控制台限制**: 限制控制台最大行数，防止内存过大
- **变量显示**: 只显示当前步骤的变量，不保存历史
- **资源释放**: 正确管理Qt对象生命周期

### 3. 渲染优化
- **行号缓存**: 行号区域使用缓存绘制
- **语法高亮**: 使用QSyntaxHighlighter，高效重绘
- **表格优化**: 使用QTableWidget虚拟化选项
